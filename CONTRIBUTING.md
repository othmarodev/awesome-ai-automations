# Contributing

Thanks for helping keep this list useful and high-signal.

## What belongs here

A tool, bot, GitHub Action, or workflow that **automates a recurring task** — bonus points if it uses an LLM. It should be real, working, and actively maintained.

## What does not

- Abandoned projects (no commits in ~12 months).
- Generic "AI platforms" with no specific automation use case.
- Your own unreleased side project (ship it first).

## Entry format

One line, alphabetical-ish within its category, ending with a period:

```
- [Name](https://link) - What it automates, in one sentence. 🔓
```

- Link to the **GitHub repo** for open-source tools, otherwise the homepage.
- Add the 🔓 marker **only** if the tool is open source and self-hostable.
- Keep the description under ~12 words. Say what chore it removes, not marketing fluff.

## Before you open the PR

- [ ] The entry is in the right category.
- [ ] The link works and is HTTPS.
- [ ] The description is one sentence, no trailing fluff.
- [ ] No duplicates.

CI runs a link check on every PR — broken links will block the merge.
